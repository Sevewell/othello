# やりたいこと
ベイズを活かして探索結果を次の探索の事前情報にできるシステムを実現したい。あとRustやってみたい。

# 方法
木構造とハッシュテーブルの組み合わせで叶うのでは。
- ハッシュテーブルから木構造データを生成する
- 木構造データで探索
- 木構造データからハッシュテーブルを生成

# ハッシュテーブル
インデックスは黒白を合わせた128ビット整数のmodでいける。ファイルに出力したいのでチェインではなくネクストインデックス。キーは黒白のタプル。容量の限界はあるので、幅優先のカットが要るかも。

# 木構造
生成は初期ノードからハッシュテーブルになくなるまでノード生成する処理でいける。いや、幅優先探索か。αとβを小さい実数にすることで弱い事前情報とする。
